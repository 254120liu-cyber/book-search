# 搜书网站开发方案（最终版）

## 一句话总结

做一个电子书搜索网站，部署在 Render（美国），后端直接调 Z-Library 搜索，用户搜到书后通过易支付付 2 元自动获取下载链接。总成本 0 元。

---

## 一、技术架构

```
同学手机浏览器
    ↓
book-search.onrender.com (Render 免费服务器，美国)
    ↓
Flask 后端 (Python)
    ↓               ↓
Z-Library (直连)   易支付 (收款回调)
```

- Render 在美国，不需要梯子就能直连 Z-Library
- cron-job.org 每 5 分钟 ping 一次，防止 Render 休眠
- 全自动支付，不需要人工确认

## 三、Render 免费套餐能支撑多少人？

Render Free Tier 配置：512MB RAM | 共享 CPU | 100GB 月流量

| 场景 | 估算 |
|------|------|
| Flask 运行内存 | ~80MB |
| 单次搜索耗时 | 2-5 秒 |
| 并发支持 | 3-5 人同时搜 |
| 每天可处理 | 500-1000 次搜索 |
| 月流量上限 | 100GB → 约 200 万页 → **绝对够用** |

**结论：刚开始推给同学，一天几十到几百次搜索，完全够用。** 如果真的做大到免费套餐撑不住（说明赚钱了），再升级付费套餐。

## 四、模块清单

### 4.1 搜索模块（后端核心）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| Z-Library 搜索 | P0 | 调 z-lib.id/s/?q=xxx，解析 HTML |
| 结果提取 | P0 | 书名、作者、格式、大小、年份、详情链接 |
| 下载链接获取 | P0 | 点击详情页提取真实下载链接 |
| 多域名自动切换 | P1 | z-lib.id 挂了自动换 z-lib.fm |
| 搜索缓存 | P1 | 同关键词 5 分钟内不重复请求 |

### 4.2 前端页面

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 搜索框 + 结果列表 | P0 | 移动端优先（同学用手机打开） |
| 结果卡片 | P0 | 书名/作者/格式标签/大小/年份 |
| 付费弹窗 | P0 | 点击"获取下载" → 弹出支付 |
| 分页 | P1 | 每页 20 条 |
| 热门搜索 | P2 | 首页展示高频搜索词 |

### 4.3 支付模块（易支付）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 生成支付链接 | P0 | 用户点"获取下载" → 后端生成 2 元支付链接 |
| 支付回调 | P0 | 易支付通知付款成功 → 返回下载链接 |
| 订单状态查询 | P1 | 用户可查看历史订单 |

### 4.4 部署运维

| 功能 | 优先级 | 说明 |
|------|--------|------|
| Render 部署 | P0 | push GitHub → 自动部署 |
| cron-job 保活 | P0 | 每 5 分钟 ping /api/health |
| 健康检查接口 | P0 | /api/health 返回 200 |
| 错误日志 | P1 | 记录搜索失败、支付异常 |
